# 🔍 TruthSync

> **"단 한 번의 클릭으로, 기사 속 따옴표에서 원본 맥락 영상까지."** > 뉴스 기사의 자극적인 인용구(따옴표)를 드래그하여, 실제 발언이 담긴 원본 영상의 정확한 타임라인(시·분·초)을 1초 만에 찾아내는 크롬 익스텐션 및 맥락 연결 서비스입니다.

<br />

## 1. 프로젝트 배경 및 문제 정의

### 🚨 포털 중심 뉴스 생태계와 "따옴표 저널리즘"
* [cite_start]**수익 모델과 자극성 심화:** 국내 디지털 신문 매출의 **75.8%**가 페이지뷰(PV) 기반 광고에 의존하고 있어, 언론사들은 트래픽 극대화를 위해 자극적인 인용구 중심의 기사를 양산하고 있습니다[cite: 4, 16].
* [cite_start]**따옴표 보도의 폭증:** 뉴스 본문 1개당 평균 인용구 수는 2016년 2.18개에서 2021년 **4.17개**로 급증했으며, 특히 대립이 치열한 정치 기사는 기사당 평균 **6.17개**의 따옴표가 밀집해 있습니다[cite: 7, 8].
* [cite_start]**맥락의 파편화와 악마의 편집:** 화자의 전체 발언 중 일부 자극적인 구절이나 특정 몇 초의 상황만 자의적으로 발췌하여 본래 취지를 왜곡하는 프레이밍 문제가 심각합니다[cite: 3, 4].
* [cite_start]**교차 검증의 시간적 장벽:** 독자가 프레임 왜곡을 의심하더라도, 몇 시간 분량의 원본 영상이나 청문회 풀영상에서 해당 발언 위치를 직접 찾아 검증하기에는 물리적 시간과 에너지가 너무 많이 소모됩니다[cite: 3, 18].

### 💡 해결 방향: 판단 권력을 독자에게 (Empower the Readers)
[cite_start]`TruthSync`는 AI가 뉴스의 참/거짓을 일방적으로 판결하는 심판자 역할을 하지 않습니다[cite: 18]. [cite_start]기사가 은폐한 전후 맥락을 확인할 수 있는 **객관적인 팩트의 경로(원본 풀영상의 타임라인)**를 즉시 제공하여, 독자 스스로 보도의 왜곡 여부를 주체적으로 판단할 수 있도록 돕습니다[cite: 18].

## 2. 주요 기능 및 서비스 시나리오

1. [cite_start]**독자 문맥 드래그 활성화:** 뉴스 기사를 읽다가 검증이 필요한 인용구(따옴표) 영역을 마우스로 블록 지정합니다[cite: 10, 18].
2. [cite_start]**크롬 익스텐션 실행:** 우클릭 또는 익스텐션 아이콘 클릭 한 번으로 `TruthSync`를 구동합니다[cite: 10, 18].
3. [cite_start]**실시간 원본 맥락 연동:** 플랫폼 이동이나 추가 검색 없이, 브라우저 내 오버레이 형태로 해당 발언의 전후 맥락이 담긴 유튜브 원본 영상과 자막이 매칭되어 재생됩니다[cite: 8, 11].
4. [cite_start]**독자의 능동적 식별:** 화자의 표정, 어투, 앞뒤 인과관계 및 단서 조항을 직접 확인하여 언론사의 고의적 짜깁기와 편향적 낙인찍기를 상시 견제합니다[cite: 3, 8].

<br />

## 3. 시스템 아키텍처 및 핵심 구현 단계

[cite_start]`TruthSync`는 하드웨어 인프라(Client Extension)와 LLM 및 시맨틱 매칭 알고리즘 기반의 데이터 파이프라인(Server)이 유기적으로 결합되어 작동합니다[cite: 14, 20].


```

[사용자 드래그 문장] ➔ [1. 기사 메타데이터 수집 & 출처 추론] ➔ [2. 유튜브 스마트 검색 & 랭킹]
⬇
[최종 타임라인 제공] ⮠ [4. 고차원 시맨틱 벡터 매칭] ⮠ [3. 하이브리드 자막 시퀀스 대조]

```

### 🛠️ Step 01. 기사 정보 수집 및 출처 추론
* [cite_start]사용자가 드래그한 문장과 해당 웹페이지의 메타정보(기사 제목, 작성 일자, 언론사명, URL)를 일괄 수집하여 서버로 전송합니다[cite: 13, 14].
* [cite_start]수집된 데이터를 **Claude API**에 전달하여, 발언이 이루어진 실제 오리지널 행사명 또는 원본 후보 영상의 타이틀을 프롬프트 엔지니어링을 통해 추론합니다[cite: 13, 14].

### 📊 Step 02. 유튜브 스마트 검색 및 신뢰도 랭킹 (Scoring)
* [cite_start]추론된 타이틀을 기반으로 YouTube API 쿼리를 송신하여 20~30개의 후보군 영상 JSON 데이터를 추출합니다[cite: 11, 13].
* [cite_start]단순히 조회수나 인기순 정렬을 배제하고, 팩트체크 목적에 특화된 가중치 알고리즘을 적용합니다[cite: 11, 14].
  * [cite_start]**채널 가중치:** 국회방송, 지상파 보도국 등 공신력 있는 공식 아카이브 채널에 우선순위 부여 [cite: 11, 14]
  * [cite_start]**길이 가중치:** 10분 내외의 자의적 편집본을 배제하고, 1~5시간 분량의 원본 풀영상(Full Video) 우대 [cite: 11, 14]
  * [cite_start]**최종 점수 산출:** `[채널 점수] + [길이 점수] + [유튜브 순위 점수] + [제목 일치 보너스]` [cite: 14]

### 🔀 Step 03. 하이브리드 세그먼트 자막 대조 연산
* [cite_start]선출된 영상들의 자동 생성 및 공식 자막 스크립트 대본을 클라우드에서 로드합니다[cite: 14, 15].
* [cite_start]사용자 드래그 문장과 전체 자막 데이터 간의 하이브리드 유사도 비교 알고리즘을 수행합니다[cite: 12, 14].
  * [cite_start]**단어 겹침 (Overlap):** 명사 및 핵심 도메인 키워드의 일치율 분석 [cite: 12, 14]
  * [cite_start]**문장 구조 (Sequence):** 형태소 배열 뼈대 및 발화 순서의 흐름 구조 검증 [cite: 12, 14]

### 🎯 Step 04. 벡터 임베딩 기반 최종 후보 선출
* [cite_start]추출된 후보 자막 문장 세그먼트들을 **Gemini 언어 모델**로 전송하여 고차원 시맨틱 벡터 임베딩(Vector Embedding)을 생성합니다[cite: 13, 14].
* [cite_start]**코사인 유사도(Cosine Similarity)** 연산을 수행하여 맥락 및 의미상 가장 일치하는 상위 3개의 구간을 최종 선출합니다[cite: 13, 14].
* [cite_start]일치율이 가장 높은 자막 문장의 타임스탬프(시·분·초)를 크롬 익스텐션 UI를 통해 사용자에게 즉각 매핑하여 제공합니다[cite: 13, 14].

## 4. 기술 스택 (Tech Stack)

* [cite_start]**Client:** JavaScript, HTML5, CSS3 (Chrome Extension Architecture) [cite: 17]
* [cite_start]**Server:** Python / Node.js [cite: 17]
* [cite_start]**AI & LLM:** Claude API (출처 및 키워드 추론), Gemini API (Vector Embedding & 시맨틱 매칭) [cite: 13, 14]
* [cite_start]**Database & API:** YouTube Data API (자막 및 메타데이터 파싱) [cite: 13, 14]
* [cite_start]**Algorithm:** Cosine Similarity, Hybrid Overlap-Sequence Matching NLP Logic [cite: 13, 14]

### ✨ 기대 효과
* [cite_start]**주도적 맥락 검증으로의 전환:** 언론이 일방적으로 짜깁기한 따옴표를 그대로 수용하던 방식에서 벗어나, 독자에게 능동적인 공론장 감시 주체성을 부여합니다[cite: 14, 18].
* [cite_start]**정치적/사회적 프레이밍 견제:** 실시간으로 누락된 단서 조항이나 전제 조건을 대중이 직접 식별함으로써 언론사의 악의적인 편향적 낙인찍기 관행을 상시 제어합니다[cite: 14, 15].
